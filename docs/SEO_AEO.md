## 1. Page-type taxonomy
`/product/[slug]`, `/category/[slug]`, `/brand/[slug]`, `/scent/[slug]`, `/best/[slug]`,
`/compare/[slug]`, `/guide/[slug]`, `/article/[slug]` — each with localized variants under
`/en` and `/zh` (Thai no prefix). Slugs are per-locale.

## 2. Required metadata (every key page)
Meta title · meta description · canonical (self, localized) · Open Graph (title/description/
image/type/locale) · Twitter card · breadcrumb · robots directives · `hreflang` alternates
(published locales only + `x-default` → Thai) · localized FAQ · AEO summary · pros/cons ·
best-for / not-for · JSON-LD. Implement via Next.js `generateMetadata` per route, pulling from
`*_translations` + `seo_metadata`.

## 3. JSON-LD schema usage
| Page | Schema |
|------|--------|
| Product detail | `Product` + `Review` + `AggregateRating` + `FAQPage` + `BreadcrumbList` |
| Category / Best | `ItemList` + `BreadcrumbList` (+ `FAQPage`) |
| Brand / Scent | `ItemList` + `BreadcrumbList` |
| Article / Guide | `Article` + `FAQPage` + `BreadcrumbList` |
| Compare | `ItemList` + `FAQPage` + `BreadcrumbList` |
| Global | `Organization` (logo, name หอมฉลุย, sameAs) |

Notes: `Product.offers` reflects merchant links (use `AggregateOffer` across merchants with
`priceCurrency`, low/high price); set `inLanguage` to the page locale; never invent ratings —
`AggregateRating` only when real reviews exist; affiliate offers marked appropriately.

## 4. AEO (Answer Engine Optimization)
Each key page carries a machine-friendly answer block:
- **3–5 line summary** answering "what is it / is it good / who is it for".
- **FAQ** with direct, concise answers.
- **Best for / Not for**.
- **Approximate price** and **where to buy** (merchants).
- **Pros / cons**.
Keep answers self-contained and factual; mirror them in `FAQPage` JSON-LD. Store the AEO
summary in `*_translations.aeo_summary` (per locale).

## 5. Sitemaps & robots
- `/sitemap.xml` is a sitemap index → `/sitemap-th.xml`, `/sitemap-en.xml`, `/sitemap-zh.xml`.
- Each localized sitemap lists **only published** localized URLs; include `xhtml:link`
  hreflang alternates.
- Generated by the worker on schedule and on publish.
- `robots.txt`: allow public pages; disallow `/admin`, `/api`, `/go`; reference sitemap index.

## 6. Multilingual SEO rules
See `I18N_RULES.md §7` and `docs/MULTILINGUAL_SEO.md`. Never emit hreflang/sitemap entries for
unpublished/untranslated localized pages; never show Thai content on `/en` or `/zh`.

## 7. Health & governance
The backoffice SEO Manager flags missing titles/descriptions/FAQ/schema, duplicate slugs, and
canonical errors, and computes a per-locale **SEO health score**. Broken merchant links are
detected by the link-checker job and excluded from `Product.offers`.

## 8. Performance-as-SEO
Server-rendered HTML, fast LCP, image optimization, stable layout (no CLS from badges/cards),
and clean semantic markup. Tracking must not block rendering or navigation.
